Gedacht ist, dass das Tool für Benutzer auf ihren persönlichen Desktops verteilt wird. Benutzer sollen geschult werden, bei Verbindungsproblemen, noch vor dem Supportruf, das Programm auszuführen – durch gewonnene Informationen werden Einige eventuell selbständig auf die Problemlösung kommen.

Das Programm ermittelt aktuelle IP-Adresse, zwingt den Computer die dynamische Adresse erneut anzufordern, und diagnostiziert anhand des festgelegten Werts den Status des Netzwerks. Hierbei wird davon ausgegangen, dass die richtige Adresse vom DHCP-Server, aus privaten Netzwerkbereichen (Class ABC) vergeben wird. Wenn das lokale Netzwerk funktioniert, wird außerdem versucht, eine Verbindung zur internem Server und dann zu externen Ressource herzustellen. Dadurch wird festgelegt, ob der interne Server erreichbar ist und ob ein Zugriff auf das Internet besteht – das wird dem Benutzer mitgeteilt. Am Ende wird noch ein kurzes Systeminfofenster eingeblendet.
